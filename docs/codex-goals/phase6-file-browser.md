# Codex Goal: Phase 6 — サイドバー内蔵ミニファイルブラウザ

## ゴール

サイドバーに「files」モードを追加し、**外部ツール（yazi等）なしで**フォルダを
クリックで潜り、ファイルをクリックでプレビュー（→既存の[編集]でエディタ）できるようにする。
Windows 配布時に gototerm 単体で「見る・潜る・開く」が完結することが狙い。

**やらないこと（重要）**：ファイル操作（コピー/移動/削除/リネーム/新規作成）は一切作らない。
それは yazi 等の領分。ここは「見る・選ぶ・開く」だけ。

## 前提（先に読むこと）

- `src/sidebar.rs` … `RowAction` / `on_click` / `SidebarRequest` / rebuild のレイアウト機構
- `src/preview.rs` … クリック→ピン留めプレビューの既存経路
- `docs/codex-goals/phase3-live-preview.md` の絶対制約（UIスレッドでブロックする I/O 禁止・fmtは触ったファイルのみ）

## 追加の絶対制約（今日の起動クラッシュからの教訓）

- **config に新フィールドを足す場合は必ず `#[serde(default)]` を付ける**か、非空のデフォルトを持たせること。
  configクレートは空配列のデフォルトをシリアライズで落とすため、既存ユーザーの設定ファイルで
  起動時 panic する（今日 `editor` で実際に起きた）。

## 実装内容

### 1. サイドバーのモード切替

- `Sidebar` に `mode: SidebarMode`（`Changes` | `Files`、初期値 `Changes`）を追加。
- ヘッダ（workspace/git 行の下）に切替行を1行追加：
  ```
   [changes]  files        ← 現在モードを白背景等で強調。クリックで切替
  ```
  行単位クリックの既存機構でよい（クリックのたびにトグル）。
- `Changes` モードは現行表示そのまま（changed files ＋プレビュー）。
- `Files` モードは下記のブラウズ表示＋プレビュー（プレビュー部は共通）。

### 2. Files モード（ブラウズ）

- `browse_dir: PathBuf`（初期値 = watcher root と同じ cwd）。cd 追従で root が変わったら
  browse_dir もリセットする。
- 表示：
  ```
   files  ~/work/programs/toyterm     ← browse_dir（末尾省略表示）
     ../                              ← 親へ（ファイルシステムルートでは出さない）
     docs/
     src/
     Cargo.toml
     README.md
     … ほか 12 件
  ```
  - ディレクトリ先頭（末尾 `/`・Color::Blue）→ ファイル（White）の順、それぞれ名前順。
  - 隠しファイル（`.` 始まり）は表示しない。
  - 表示件数はプレビュー領域を潰さないよう最大 15 行。超過は「… ほか N 件」。
  - **マウスホイールでリストをスクロール**できるようにする（超過分にアクセスする手段）。
    `Sidebar` にホイール通知の口（例 `on_scroll(delta)`）を追加し、`multiplexer.rs` の
    「サイドバー内ホイールを握り潰す」箇所から委譲する。Changes モードでは何もしない。
- クリック動作（`RowAction` に追加）：
  - `../` またはディレクトリ行 → `browse_dir` を移動して一覧を更新
  - ファイル行 → 既存のピン留めプレビュー（`[編集]` もそのまま効く）
- **一覧の読込（read_dir）はバックグラウンド**で行う（preview と同じ spawn ＋ try_recv パターン。
  ネットワークマウントのディレクトリでUIを固めない）。読込中は「(読込中…)」を表示。
  エントリ数は最大 500 件で打ち切り（ソートしてから切る）。
- ディレクトリ移動は watcher / changed files に影響させない（ブラウズは独立した閲覧機能）。

### 3. 純関数＋テスト

```rust
/// read_dir の結果を表示順に並べる：dir優先→名前順。隠しファイル除外。max件で打ち切り。
fn sort_entries(entries: Vec<(String, bool /*is_dir*/)>, max: usize) -> Vec<(String, bool)>
```

- テスト：dir優先 / 名前順 / 隠し除外 / 打ち切り / 空ディレクトリ。
- スクロールオフセットのクランプ（0..=len-visible）も純関数にしてテスト。

## 制約

- 新規依存なし（std のみ）。
- ファイル操作機能は追加しない（右クリックメニュー等も不要）。
- 既存の Changes モードの挙動・レイアウトを変えない（モード切替行が1行増えるのみ）。
- 既存コードの無関係な整形禁止。cargo fmt は自分が触ったファイルのみ。
- コメントは日本語で「なぜ」を書く。

## 完了条件（この順で検証）

1. `cargo test` 全パス（`sort_entries` / スクロールクランプの新規テスト含む）。
2. `cargo build --release` 成功、新規警告なし。
3. 手動確認手順を `docs/codex-goals/phase6-verify.md` に出力：
   サイドバー表示 → 切替行クリックで files モード → フォルダをクリックして潜る →
   `../` で戻る → ファイルをクリックでプレビュー → [編集] で nvim →
   ホイールで長い一覧をスクロール → changes に切り替えると従来表示 →
   ネットワーク上の巨大フォルダを開いても画面が固まらない。
